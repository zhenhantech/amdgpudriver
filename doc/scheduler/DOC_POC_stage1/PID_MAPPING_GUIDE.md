# PID映射问题：AMD日志 vs ftrace

**问题**: 容器内PID (157868) ≠ 主机PID (3934101)

## 原因：Docker PID Namespace

```
容器内 (AMD日志):  PID 157868 ← 容器的PID空间
                      ↓ 映射
主机 (ftrace):     PID 3934101 ← 内核看到的真实PID
```

## 解决方案：通过时间戳关联 ✅

### 示例
```
AMD日志:  177770.497秒 | acquireQueue (PID 157868)
ftrace:   177770.631秒 | kfd_open (PID 3934101)
         → 延迟133ms，是同一个进程的操作！
```

## 关联方法

1. **不要用PID匹配** ❌
2. **用时间戳关联** ✅
3. **用correlate_logs.py工具** ✅

## 关键点

- ✅ 这是Docker的正常行为
- ✅ 不影响分析（时间戳一致）
- ✅ KFD自动处理映射
- ✅ 我们的分析已经处理了这个问题

## 已验证

| 项目 | 容器内 | 主机 |
|------|--------|------|
| PID | 157868 | 3934101 |
| 时间范围 | 177770~177777秒 | 177770~177777秒 ✅ |
| 事件序列 | acquireQueue→使用 | kfd_open→创建→使用 ✅ |

**结论**: 通过时间戳完美关联，PID不匹配不是问题！
